任务依赖规则

示例：A 是 B 的前置任务，B 是 C 的前置任务，C 为末尾任务。

A 计划完成：7月25日 18:00
B 计划完成：7月27日 18:00
C 计划完成：7月29日 18:00
C DDL：      7月30日 18:00

当 A 改为 7月27日 18:00（推迟2天）：
B 自动改为 7月29日 18:00
C 自动改为 7月31日 18:00
由于 C 超过 DDL，C 自动置顶并显示“需重新排期”。

注意：
- 只在前置任务的计划完成时间向后调整时顺延；提前不会自动拉前后续任务。
- 后续任务未填写计划完成时间时，不自动生成时间。
- 末尾任务必须填写计划完成时间与 DDL。
- 导入时，多个前置任务ID使用 | 分隔。
